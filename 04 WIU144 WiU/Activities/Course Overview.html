Instructions

Course Objective

Show students there is more to website design then nav bars and graphics. It's about creating intuitive "familiar" interfaces that are easy and enjoyable to use from the users perspective. Strive to support the users needs and wants by illuminating the main intent of any web page or interface.

Pre requisite Knowledge

Students will need only the knowledge of Illustrator and Photoshop, a basic understanding of web terminologies and the desire to create better experiences for users of web interfaces.

    Objective Knowledge the student will learn in WiU
  • The basis for Interaction design
  • Designing for the way we interact with things
  • Applying affordances effectively to any artifact
  • What Usability is and why it's important
  • It makes our interactions easier and more enjoyable
  • HCI is a marriage of Disciplines
  • Computer Science, Behavioral Science and Design (interface)
  • When combined they define how humans and computers will interact
  • Web Interfaces 3 types to support our main intent
  • Promotional
  • Tool or Technology
  • Educational
    •
  • Design Patterns solve design problems
  • By designing for user needs make the main intent of any Ui obvious
  • To improve the usability example: pagination
How WiU fits within the entire WDDBS degree

WiU is one of 2 design courses in the web program that is specifically focused on the design considerations when creating any web interface and its Usability. WiU - Web interface & Usability, (woo) that's nice.

Interaction / Usability / Interfaces / Patterns

Interactive Design Principles

Visibility, feedback, constraints, mapping, consistency, affordance

Usability Design Principles

Learnability, efficiency, memorability, errors, satisfaction

Interfaces
  • Promotional - lots of imagery, video, animations, minimal text designed to promote or sell
  • Tool / Tech - few images, lots of copy (text), designed to retrieve information fast
  • Educational - sequenced navigations used to organize and present prerequisite knowledge
Design Patterns

User Behavior - Safe exploration, instant gratification, satisficing, changes in midstream, deferred choices, habituation, spatial memory

Information Architecture - Two panel selectors, canvas plus palette, one window drill downs, wizards, extras on demand

Navigation - clear entry points, global, hub & spoke, pyramid, bread crumbs, annotated scrollbars, color coded sections, animated transitions, escape hatch

Page Layout - visual framework, center stage, titled sections, card stack, right/left alignment, diagonal balance, responsive enabling, responsive disclosure

Interactivity - button groups, action panels, prominent done/submit buttons, progress indicators

Complex Data - overview plus detail, data tips, dynamic queries, row striping, sortable table, cascading lists, multi y graphs, small multiples

Forms - structure format, input hints/prompts, auto completion, illustrated choices, defaults, errors

Editors - Edit in place, smart selection, composite selection, spring loaded mode, magnetism
